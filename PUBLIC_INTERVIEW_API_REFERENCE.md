# Production-Ready Public Interview System - API Reference

## Public Interview Session Endpoints

These endpoints are for **public candidates** (no authentication required).

---

## Session Management Endpoints

```
    "difficultyLevel": "medium",
    "numberOfQuestions": 5

```

---

### 2. Start Interview Session (with Session Locking)
```
POST /api/interview/session/:token/start

Request Body:
{
  "candidateName": "John Doe",
  "candidateEmail": "john@example.com",
  "sessionFingerprint": "fp_1234567890_abc123"  // Generated by frontend
}

Success Response: 201 Created
{
  "success": true,
  "message": "Interview session started",
  "data": {
    "sessionId": "interview_id_123",
    "sessionLockId": "uuid-generated-lock-id",  // IMPORTANT: Store in sessionStorage
    "questions": ["Question 1", "Question 2"],
    "numberOfQuestions": 5,
    "jobRole": "frontend",
    "experienceLevel": "senior",
    "interviewType": "technical",
    "difficultyLevel": "medium"
  }
}

Error: 409 Conflict (Session Already Locked)
{
  "success": false,
  "message": "This interview session is already in progress by another candidate...",
  "code": "SESSION_LOCKED"
}
```

---

### 3. Resume Interview Session (After Refresh)
```
GET /api/interview/session/:interviewId/resume?sessionLockId=uuid-lock-id

Success Response: 200 OK
{
  "success": true,
  "message": "Interview session resumed",
  "data": {
    "sessionId": "interview_id_123",
    "currentQuestionIndex": 2,           // Restored from server
    "answers": {
      "0": "My answer to Q1",
      "1": "My answer to Q2"
    },
    "transcript": "saved transcript text",
    "questions": ["Q1", "Q2", "Q3"],
    "status": "in-progress"
  }
}

Error: 410 Gone (Session Expired)
{
  "success": false,
  "message": "Interview session has expired",
  "code": "SESSION_EXPIRED"
}

Error: 401 Unauthorized (Wrong Lock)
{
  "success": false,
  "message": "Invalid session"
}
```

## Error Codes Reference

| Code | HTTP | Meaning | User Message |
|------|------|---------|--------------|
| SESSION_LOCKED | 409 | Another candidate is using this session | "Interview already in progress" |
| SESSION_EXPIRED | 410 | 30+ minutes have passed | "Session expired, start new interview" |
| INVALID_SESSION | 401 | Wrong sessionLockId or missing | "Invalid session" |
| INTERVIEW_COMPLETED | 400 | Can't modify completed interview | "Interview already submitted" |
| NETWORK_ERROR | varies | Connection failed | Shows with "Retrying..." |

## Frontend Integration Guide

### 1. Session Initialization

```javascript
// Generate fingerprint
const sessionFingerprint = `fp_${Date.now()}_${Math.random().toString(36).substr(2, 9)}`

// Start interview
const response = await fetch(`/api/interview/session/${token}/start`, {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({
    candidateName,
    candidateEmail,
    sessionFingerprint
  })
})

const data = await response.json()

// Store in sessionStorage (session-specific, cleared on browser close)
sessionStorage.setItem('sessionLockId', data.data.sessionLockId)
sessionStorage.setItem('interviewId', data.data.interviewId)

// Store in localStorage (persistent across refresh)
localStorage.setItem('currentInterview', JSON.stringify(data.data))
```

### 2. Resume After Refresh

```javascript
// Try to resume from server
const sessionLockId = sessionStorage.getItem('sessionLockId')
const interviewId = sessionStorage.getItem('interviewId')

if (sessionLockId && interviewId) {
  const response = await fetch(
    `/api/interview/session/${interviewId}/resume?sessionLockId=${sessionLockId}`
  )
  
  if (response.ok) {
    const data = await response.json()
    // Restore: currentQuestionIndex, answers, transcript
    setCurrentQuestionIndex(data.data.currentQuestionIndex)
    setAnswers(data.data.answers)
  }
}
```

### 3. Save Progress After Each Answer

```javascript
// After each answer is recorded
await fetch(`/api/interview/session/${interviewId}/progress`, {
  method: 'PUT',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({
    sessionLockId,
    currentQuestionIndex,
    answers,
    transcript
  })
})
```

### 4. Submit with Retry Logic

```javascript
import { retryInterviewSubmission } from '../services/retryService'

// Use retry wrapper for submission
await retryInterviewSubmission(
  async () => {
    const response = await fetch(
      `/api/interview/session/${interviewId}/submit`,
      {
        method: 'POST',
        headers: { 'Content-Type': 'application/json' },
        body: JSON.stringify({
          interviewId,
          sessionLockId,
          questions,
          answers,
          // ... other fields
        })
      }
    )
    
    if (!response.ok) throw new Error('Submission failed')
    return response.json()
  },
  (message, isRetrying) => {
    if (isRetrying) {
      showToast(message) // "Retrying... (Attempt 1/3)"
    }
  }
)
```

## Session Lifecycle

```
1. Candidate opens interview link
   ↓
2. Enters name & email
   ↓
3. Clicks "Start Interview"
   ↓
4. Frontend generates fingerprint
   ↓
5. POST /session/:token/start
   → Server locks session with fingerprint
   → Returns sessionLockId
   ↓
6. Frontend stores sessionLockId in sessionStorage
   ↓
7. Candidate answers questions
   ↓ (After each answer)
8. PUT /session/:id/progress
   → Server saves snapshot
   ↓
9. If refresh/close → Browser closes, sessionStorage cleared
   ↓
10. Candidate reopens link
    ↓
11. Frontend has sessionFingerprint from sessionStorage
    ↓
12. GET /session/:id/resume
    → Server checks if lock matches fingerprint
    → Returns saved state
    ↓
13. Frontend restores answers, current question
    ↓
14. Candidate continues where they left off
    ↓
15. Submit all answers
    ↓
16. POST /session/:id/submit
    → Server evaluates
    → Generates submissionId
    → Stores evaluation
    ↓
17. Duplicate submission
    ↓
18. POST /session/:id/submit (same answers)
    → Server checks submissionId exists
    → Returns 200 with cached evaluation
    → No re-evaluation
```

## Timeout Scenarios

### Server-Side Timeout (Automatic)

```
Session starts: 10:00 AM
Candidate stops answering at: 10:15 AM
Server cron runs every 5 minutes

10:20 AM - No activity for 5 min (under threshold)
10:25 AM - No activity for 10 min (under threshold)
...continues...
10:30 AM - Cron checks: sessionStartedAt was 10:00 AM
         - 30 minutes have passed
         - sessionStatus = "expired"
         - Interview status = "failed"
```

**Result**: Session marked as failed, cannot resume, must start new interview.

### Client-Side Timeout (Prevention)

```javascript
// Frontend sets 30-min timer
const timeout = setTimeout(() => {
  toast.error('Interview session expired due to inactivity')
  navigate(`/interview/session/${token}`) // Redirect to start
}, 30 * 60 * 1000)

// Reset timer on user activity (click, keypress)
document.addEventListener('click', resetTimeout)
```

## Session Locking Example

### Browser 1 - First Candidate

```
1. POST /session/:token/start
   {sessionFingerprint: "fp_1234_abc"}
   ↓
2. Server creates interview
   sessionStatus = "locked"
   sessionLockedBy = "fp_1234_abc"  ← LOCKED!
   ↓
3. Returns 201 Created
```

### Browser 2 - Second Candidate (Same Link)

```
1. POST /session/:token/start
   {sessionFingerprint: "fp_5678_xyz"}
   ↓
2. Server checks for existing lock
   Finds: sessionStatus = "locked", sessionLockedBy = "fp_1234_abc"
   Checks time: started 2 minutes ago (under 30-min timeout)
   ↓
3. Returns 409 Conflict
   Message: "Interview already in progress by another candidate"
```

### After 30 Minutes

```
1. Cron job runs (every 5 min)
   Finds: sessionStatus = "locked", sessionStartedAt before cutoff
   ↓
2. Updates interview:
   sessionStatus = "expired"
   status = "failed"
   ↓
3. If first candidate opens link again
   POST /session/:token/start (same or different fingerprint)
   ↓
4. Server finds old locked session is expired
   Releases lock
   Creates new session with new lock
```

## Debugging Checklist

### If Session Won't Lock
- [ ] `sessionFingerprint` is being sent in start request
- [ ] Server is receiving it
- [ ] Interview document has `sessionLockedBy` field
- Check MongoDB: `db.interviews.findOne({_id: interview_id})`

### If Resume Not Working
- [ ] `sessionLockId` stored in sessionStorage
- [ ] `interviewId` available
- [ ] Resume endpoint returns 200 (not 401 or 410)
- [ ] `currentQuestionIndex` and `answers` in response

### If Submission Fails
- [ ] `sessionLockId` is correct
- [ ] Interview not already completed (`submissionId` not set)
- [ ] Session not expired (check cron logs)
- [ ] Retry logic triggered (check for "Retrying..." message)
- [ ] Check MongoDB for submission errors

### If Dashboard Not Updating
- [ ] Admin dashboard auto-refresh running (check every 30 sec)
- [ ] New interview has `isTemplateBasedInterview: true`
- [ ] Status correctly set to "completed"
- [ ] Evaluation object populated with score

## Production Monitoring

### Key Metrics to Track

```javascript
// Active sessions right now
db.interviews.countDocuments({ sessionStatus: "locked" })

// Sessions expired today
db.interviews.countDocuments({
  sessionStatus: "expired",
  updatedAt: { $gte: today }
})

// Resume rate (questions resumed, not started)
db.interviews.countDocuments({ currentQuestionIndex: { $gt: 0 } })

// Submission success rate
const completed = db.interviews.countDocuments({ status: "completed" })
const total = db.interviews.countDocuments({ isTemplateBasedInterview: true })
const successRate = (completed / total) * 100
```
